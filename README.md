## ⚙️ Как это работает

Система использует **автоэнкодер** — нейронную сеть, которая учится сжимать и восстанавливать данные. Принцип детекции аномалии основан на ошибке реконструкции:

```
Аудио → Мел-спектрограмма → Нормализация → Автоэнкодер → MSE → Аномалия?
```

1. Модель обучается на **нормальных** звуках поезда
2. При мониторинге она пытается восстановить входной аудиосигнал
3. Если ошибка восстановления (`MSE`) превышает порог — **аномалия**

Аудио преобразуется в логарифмическую мел-спектрограмму размером `100 × 64`, которая разворачивается в вектор **6400 признаков**.

## 🧠 Архитектура модели

```
┌─────────────────────────────────────────────────────────┐
│                      АВТОЭНКОДЕР                        │
│                                                         │
│  Вход      Encoder              Decoder      Выход      │
│  (6400) → Dense(64) → Dense(32) → Dense(32) → (6400)   │
│                    ↘ Dense(16) ↗                        │
│                    [bottleneck]                         │
└─────────────────────────────────────────────────────────┘

  Активация: ReLU (скрытые слои), Sigmoid (выходной слой)
  Функция потерь: MSE  |  Оптимизатор: Adam
```

**Требования:** Python 3.8+

```bash
git clone https://github.com/your-username/train-anomaly-detector.git
cd train-anomaly-detector
pip install tensorflow librosa scikit-learn sounddevice joblib numpy
```

> **Windows:** При ошибках с `sounddevice` установите [PortAudio](https://www.portaudio.com/)

## 🚀 Использование

### Шаг 1 — Подготовка данных

Поместите обучающие `.wav` файлы (нормальные звуки поезда) в папку `data/ToyTrain/`:

```
data/
└── ToyTrain/
    ├── normal_01.wav
    ├── normal_02.wav
    └── ...
```

Запустите предобработку:

```bash
python preprocess.py
```

Будут созданы `X_train_scaled.npy` и `scaler.pkl`.

### Шаг 2 — Обучение модели

```bash
python train_model.py
```

Сохраняет `anomaly_detector.keras` и `threshold.txt` (порог = 95-й перцентиль MSE на обучающих данных).

### Шаг 3 — Мониторинг в реальном времени

```bash
python detect_live.py
```

Система начнёт слушать микрофон и анализировать звук. Нажмите **Enter** или **Ctrl+C** для выхода.

### Настройка порога (опционально)

Если система слишком чувствительна или пропускает аномалии:

```bash
python fix_threshold.py
```

Или отредактируйте `threshold.txt` вручную. Текущее значение: **3.5**

## 📁 Структура проекта

```
train-anomaly-detector/
│
├── 📄 preprocess.py           # Предобработка WAV → мел-спектрограмма
├── 📄 train_model.py          # Обучение автоэнкодера
├── 📄 detect_live.py          # Детектирование аномалий в реальном времени
├── 📄 fix_threshold.py        # Ручная настройка порога
│
├── 🧠 anomaly_detector.keras  # Обученная модель (~10 MB)
├── ⚖️  scaler.pkl              # Нормализатор данных (StandardScaler)
└── 📊 threshold.txt           # Пороговое значение MSE
```

## 🔧 Параметры

| Параметр | Значение | Описание |
|---|---|---|
| `SAMPLE_RATE` | 22 050 Гц | Частота дискретизации |
| `N_MELS` | 64 | Количество мел-фильтров |
| `DURATION` | 2 сек | Длина анализируемого окна |
| `N_FFT` | 2048 | Размер окна FFT |
| `HOP_LENGTH` | 512 | Шаг между фреймами |
| `SEQUENCE_LEN` | 100 | Фреймов в последовательности |
| `INPUT_DIM` | 6 400 | Размерность входа модели |
| `threshold` | **3.5** | Порог аномалии (MSE) |
